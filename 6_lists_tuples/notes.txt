Short Circuiting
---

- short circuiting occurs when using 'and' operators

- if the first value is FALSE --> the second statement is short circuited (ignored)

if 1 < 2 and spam[0] == 'cat':

- 1<2 = false therefore spam[0] == 'cat' is ignored
--> spam[0] == 'cat' would have caused an error (since cat doesnt exist)

- but BECAUSE OF SHORT Circuiting, we are GOOD! because python ignored the second conditional statement

Tuples
---

- 2 main differences from list
    - defined using () instead of []
    - they are IMMUTABLE (cannot modify, append, or remove values)


- used when you DO NOT WANT the order of values to change
- slightly faster than lists because of optimizations

---
References
---

- lists and tuples are usually PASSED BY References

--> you are passing a REFERENCE to the list in memory

--> you aren't creating a clone, you are actively modifying the OG list in memory

spam = [1,2]
eggs = spam

- eggs is a REFERENCE to [1,2] in memory
- the same value as spam (they are the same address value)
